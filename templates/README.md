# TEMPLATE

README.md

Version: 1.0.0

---

# Purpose

This template generates the project's root README.md.

The README is the entry point for every generated game project.

It should enable a new developer to understand the project within five minutes.

---

# Output Structure

The generated README must contain the following sections in order.

---

# Project Title

Display:

Game Name

One-line Description

Current Version

Development Status

Example:

# Wool Workshop

A cozy yarn puzzle game for WeChat Mini Games.

Version: 0.1.0

Status: MVP Development

---

# Project Overview

Explain:

- What is the game?
- Why does it exist?
- What makes it unique?

Maximum:

200 words.

---

# Core Features

- Description

# Generation Rules

The README should:

Be understandable by new developers.

Avoid implementation details.

Avoid duplicated documentation.

Link to other documents when necessary.

Keep total length under 1200 lines.

---

# Validation

Before completion verify:

✓ Project title exists

✓ Overview exists

✓ Technology stack exists

✓ Directory tree exists

✓ Document index exists

✓ Getting Started exists

✓ Contribution exists

✓ No placeholder sections remain except Contact and License

If validation fails,

regenerate the missing section.

---

# Completion Criteria

The generated README should allow a developer to:

Understand the project.

Clone the repository.

Run the project.

Locate documentation.

Begin contributing.

without additional explanation.
